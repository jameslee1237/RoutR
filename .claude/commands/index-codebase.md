# Index Codebase

Generates `FEATURE_INDEX.md` — the cheap-lookup map `prepare` consumes instead of re-scanning every run.

## When to (not) invoke

| Run | Skip |
| --- | --- |
| New top-level area merged | Per-ticket (use `prepare` — it consumes, doesn't regenerate) |
| `prepare` reports index stale (>30d) | Trivial single-file edits |
| Quarterly maintenance | During active `augmented-coding` (file lock) |
| Large refactor moves/renames feature dirs | |

## Pre-flight

1. `src/` exists with feature subdirs
2. Git tree clean (output is committed; mixing WIP is noisy)
3. `documents/specs/` exists (default). Override via `--specs-path=<path>`; `--specs-path=skip` bypasses Phase 3.
4. If existing `FEATURE_INDEX.md` < 7 days old → ask "기존 인덱스가 최근에 생성됐습니다 ({DATE}). 다시 생성할까요?"

## Phase 1 — Discover feature areas

Use Glob (not bash `ls`). Scan: `src/components/` (canonical naming — user-facing), `src/api/`, `src/store/`, `src/hooks/`, `src/app/`.

For naming mismatches across layers (e.g., `api/organization` vs `components/organization-manage`), record both names in the entry.

**Skip:** `components/common/`, `components/core-ui/` (→ Shared Infrastructure); `node_modules`, `.next`, `public`; tests (`__tests__/`, `*.test.ts`).

## Phase 2 — Map each area

Per area, gather:

1. **Components** — top-level files in `src/components/{area}/`
2. **API** — `Glob src/api/{area}/**/api.ts` (catches flat `api/vehicle/api.ts` and nested `api/auth/login/api.ts`, `dispatch/monitor`, `order/middle`, `settlement-manage/purchase`). Grep `API.{get|post|put|delete}` → list ≤5 representative endpoints + function names (augmented-coding can re-grep full lists at exec time).
3. **Stores** — Zustand stores under `src/store/{area}/**`
4. **Query keys** — grep `queryKey:` in `src/hooks/{area}/` or `src/api/{area}/query.ts`
5. **Routes** — App Router pages in `src/app/[locale]/{area}/`

**Naming inconsistencies:** if `components/{X}` exists but `api/{X}` doesn't, check `api/{X}-manage/` or `api/{X.replace('-manage','')}/`. Record as `⚠️ 컴포넌트는 {X}, API는 {Y} (네이밍 불일치)`. Known: `organization-manage`↔`organization`, `settlement`↔`settlement-manage`, `hooks/settlement-management/purchase`↔`hooks/sales-settlement` (asymmetric).

Parallel `Grep` per area; cap 50 hits each.

> 파일 직접 `Read` 시 1500줄+ 파일은 `.cursor/rules/large-file-reading.md` 전략(Strategy B: Symbol-map + Hierarchical) 적용.

## Phase 3 — Cross-reference specs

Default `--specs-path=documents/specs/`. Override via flag or `SPECS_PATH` env. `=skip` bypasses.

**Discover** — Glob `<specs-path>/*.md` excluding `SCHEMA.md`, `_template*.md`, `INDEX.md`. Read YAML front-matter (between first two `---`).

**Match (preferred — front-matter):**
1. Exact prefix match: `code_anchors.component` vs feature component path
2. Fallback: keyword match `spec_id` (snake_case) ↔ area name
3. Record full spec path + `spec_id`

**Match (legacy — no front-matter):** filename keyword (e.g., `01_로그인.md` ↔ `auth/`, `45_차량관리.md` ↔ `vehicle/`). Record path only.

No match → `Spec: 없음 (외부 또는 미작성)`. Don't fail.

**Validate `code_anchors`:** if declared path missing in current codebase → flag in `Last scan summary` as `⚠️ spec ↔ code drift: {spec_id} declares {path} which is not in the codebase`. No auto-fix — surface for human review.

## Phase 4 — Write FEATURE_INDEX.md

Output: `FEATURE_INDEX.md` at repo root. Format:

```markdown
# Feature Index

> **Generated by:** `index-codebase` skill
> **Last updated:** {ISO_DATETIME}
> **Used by:** `prepare` skill (consumes this instead of full codebase scan)
> **Refresh cadence:** on-demand (post-merge, quarterly)
> **Don't edit manually** — regenerate via `index-codebase`.

## Index

### {feature-area-name}

- **Aliases:** {if names differ across layers — e.g., "components: organization-manage, api: organization"}
- **Component path:** `src/components/{area}/`
- **API path:** `src/api/{area}/`
- **Store path:** `src/store/{...}/` or `없음`
- **Hooks path:** `src/hooks/{area}/` or `없음`
- **Routes:** `src/app/{...}` or `없음`
- **AS-IS 한 줄 요약:** {one Korean sentence}
- **주요 API 엔드포인트:**
  - `GET /api/v1/...` (함수: `getXxx`)
  - `POST /api/v1/...` (함수: `postXxx`)
- **Query keys (TanStack Query v4):**
  - `['{area}-list', filters]`
- **Zustand store slices:** {slice names or `없음`}
- **Spec:** `{spec_file_path}` (spec_id: `{SPEC_ID}`) or `없음 (외부 또는 미작성)`

### {next feature area...}

## Shared Infrastructure

> Cross-cutting concerns referenced by multiple features. `prepare` should NOT flag these as the target.

- **common** — `src/components/common/` — shared atoms/molecules/organisms (LNB, header, drawers, modals). Key exports: {5-10 most-imported}
- **core-ui** — `src/components/core-ui/` — design system primitives (button, input, badge, etc.). Key exports: {list}
- **types / constants / utils** — `src/types/`, `src/constants/`, `src/utils/` — project-wide. Local types live in feature folder `types.ts`.

## Stack snapshot (auto-pulled from package.json)

- Next.js {version}, React {version}, TypeScript {version} (⚠️ note any `@types/react` runtime gap)
- Jest {version} + Testing Library {version}
- TanStack Query {version} (⚠️ v4 — NOT v5; use `cacheTime`, not `gcTime`)
- Zustand {version}, Vanilla Extract {version}, MSW {version}

## Last scan summary

- Feature areas: {N} · component files: {N} · API functions: {N}
- Specs matched: {N} / {N_total}
- Orphans flagged: {list of API/store/hook files with no matching component or vice versa}
- Drift: {list of spec ↔ code drift entries}
```

## Phase 5 — Verify & report

1. File exists and is non-empty
2. Feature-area count ≥ 10 (sanity — typical roouty has 15+)
3. Report:
   ```
   FEATURE_INDEX.md updated.
   - {N} areas, {N} specs matched ({N_unmatched} unmatched), {N} orphans
   - Next: commit "chore: refresh FEATURE_INDEX ({DATE})"
   ```

**Don't auto-commit** — index changes deserve PR review.

## Flags

| Flag | Effect |
| --- | --- |
| `--specs-path=<path>` | Spec location. Default `documents/specs/`. `=skip` bypasses Phase 3. |
| `--dry-run` | Preview without writing |
| `--areas=<a,b>` | Partial refresh of listed areas only |

Examples: `index codebase` · `index codebase --specs-path=skip` · `index codebase --areas=auth,order --dry-run`

## Anti-patterns

- Per-ticket runs (defeats the cache — use `prepare`)
- Manual edits to FEATURE_INDEX.md (regenerated, will be overwritten)
- Including tests / generated dirs in scan (noise)
- Treating `common` / `core-ui` as feature areas (shared infra)
- Auto-committing the result (always let user review diff)

## Token economy

Full scan ~5-10k tokens (one-time per refresh). Without it, `prepare` re-discovers paths ~3-5k/ticket. Break-even: 2-3 tickets between refreshes.

## Quality bar

- [ ] FEATURE_INDEX.md exists at repo root
- [ ] Each area: component path + AS-IS summary + (≥1 endpoint OR ≥1 component file)
- [ ] Stack snapshot reflects current `package.json`
- [ ] ISO timestamp current
- [ ] Orphans/drift flagged only — no auto-fix
